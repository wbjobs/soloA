# Roguelike地牢探险游戏

一个支持2-4人联机的Roguelike地牢探险游戏，采用服务端权威架构。

## 技术栈

### 后端
- **语言**: Go 1.21+
- **Web框架**: 原生 net/http
- **WebSocket**: gorilla/websocket
- **UUID**: google/uuid

### 前端
- **游戏引擎**: Phaser 3
- **构建工具**: Vite
- **语言**: JavaScript (ES6+)

## 核心功能

### ✅ 已实现

**后端功能**:
1. **WebSocket通信**: 实时双向通信
2. **房间管理**: 创建房间、加入房间、最多4人
3. **地图生成**: 3层地牢，房间-走廊结构，程序化生成
4. **玩家系统**: 移动、属性、等级、经验值
5. **战斗系统**: 近战攻击、伤害计算、敌人反击
6. **物品系统**: 装备随机词条、物品拾取、背包管理
7. **关卡系统**: 楼层切换、难度递增

**前端功能**:
1. **主菜单**: 创建/加入房间
2. **地牢渲染**: WebGL渲染、相机跟随、缩放控制
3. **角色移动**: WASD/方向键控制
4. **战斗系统**: 空格键攻击相邻敌人
5. **UI系统**: 角色信息面板、房间信息、背包显示
6. **实时同步**: 玩家位置、状态实时更新

### 游戏特性

- **3层地牢**: 每层更大更难
- **随机装备**: 5种稀有度（普通、优秀、稀有、史诗、传说）
- **等级系统**: 击杀敌人获得经验，升级提升属性
- **多人联机**: 2-4人合作探险
- **服务端权威**: 防作弊，服务器验证所有操作

## 快速开始

### 环境要求

- **Go**: 1.21 或更高版本
- **Node.js**: 16 或更高版本（已测试 v20.9.0）
- **npm**: 8 或更高版本（已测试 10.4.0）

### 安装步骤

#### 1. 安装后端依赖

```bash
cd server
go mod download
```

#### 2. 安装前端依赖

```bash
cd ../client
npm install
```

### 运行游戏

#### 方式一：手动运行

**步骤1: 启动后端服务器**
```bash
cd server
go run main.go
```
服务器将在 `http://localhost:8080` 运行

**步骤2: 启动前端**
```bash
cd client
npm run dev
```
前端将在 `http://localhost:3000` 运行

**步骤3: 打开游戏**
- 在浏览器中访问 `http://localhost:3000`
- 第一个玩家点击"创建房间"
- 其他玩家输入房间ID，点击"加入房间"

#### 方式二：多人测试

要测试多人功能：
1. 在一个浏览器窗口创建房间
2. 复制房间ID
3. 在另一个浏览器窗口（或无痕模式）打开 `http://localhost:3000`
4. 输入房间ID和不同的玩家名称，点击"加入房间"

## 游戏机制

### 地牢结构

- **房间**: 随机大小的矩形区域，5-12格
- **走廊**: 连接房间的L形通道
- **楼梯**: 金色标记，站上去进入下一层
- **每层特色**:
  - 第1层: 60x40, 8个房间
  - 第2层: 65x43, 10个房间  
  - 第3层: 70x46, 12个房间

### 战斗系统

- **伤害公式**: `伤害 = (攻击力 - 防御力/2) ± 20%浮动`
- **最小伤害**: 1点
- **敌人反击**: 受到攻击后立即反击
- **击杀奖励**: 经验值 + 金币

### 物品系统

**装备类型**:
- 武器: 增加攻击力
- 护甲: 增加防御力和生命值
- 头盔: 增加防御力和生命值
- 靴子: 增加速度和少量防御

**稀有度**:
| 稀有度 | 颜色 | 属性倍率 |
|--------|------|----------|
| 普通 | 灰色 | 1.0x |
| 优秀 | 绿色 | 1.5x |
| 稀有 | 蓝色 | 2.0x |
| 史诗 | 紫色 | 3.0x |
| 传说 | 橙色 | 5.0x |

### 等级系统

- **升级需求**: `等级 × 100` 经验值
- **升级奖励**: 
  - 最大生命值 +20
  - 攻击力 +3
  - 防御力 +2
  - 生命值回满

## WebSocket协议

### 客户端 → 服务器

```json
// 创建房间
{"type": "create_room"}

// 加入房间
{"type": "join_room", "payload": {"roomId": "xxx", "name": "Hero"}}

// 移动
{"type": "move", "payload": {"x": 10, "y": 20}}

// 攻击
{"type": "attack", "payload": {"enemyId": "enemy_xxx"}}

// 获取状态
{"type": "get_state"}
```

### 服务器 → 客户端

```json
// 房间创建成功
{"type": "room_created", "roomId": "xxx"}

// 加入房间成功
{"type": "joined_room", "player": {...}, "players": [...]}

// 玩家加入
{"type": "player_joined", "player": {...}, "players": [...]}

// 状态更新
{"type": "state_update", "state": {...}}

// 攻击结果
{"type": "attack_result", "enemy": {...}, "state": {...}}
```

## 架构说明

### 服务端权威架构

所有游戏逻辑都在服务器端执行：
- ✅ 移动验证：检查目标位置是否可通行、是否相邻
- ✅ 战斗验证：检查攻击距离、伤害计算
- ✅ 状态同步：服务器广播状态，客户端只负责渲染
- ✅ 防作弊：客户端无法直接修改游戏状态

### 网络同步

- **状态广播**: 任何玩家状态变化时，服务器向所有连接的客户端广播
- **实时通信**: WebSocket 双向通信
- **断线重连**: 自动尝试重连（最多5次）

## 开发建议

### 添加新物品类型

1. 在 `server/internal/game/types.go` 中添加新的 ItemType
2. 在 `server/internal/game/item_generator.go` 中添加生成逻辑
3. 在 `client/src/scenes/GameScene.js` 中调整渲染颜色

### 添加新敌人

1. 在 `server/internal/game/map_generator.go` 的 `enemyTypes` 中添加
2. 调整 `generateEnemies` 函数中的属性配置

### 自定义地图

修改 `MapGenerator` 类中的参数：
- `minRoomSize` / `maxRoomSize`: 房间大小范围
- `numRooms`: 每层房间数
- 地图尺寸公式: `60 + level*5` x `40 + level*3`
